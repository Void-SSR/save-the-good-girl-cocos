# HomeScene 接入清单

这份清单用于把当前已经写好的首页控制器正式接入 Cocos 编辑器场景。

## 当前现状

已经就绪：

- 首页运行时控制器：
  - `assets/scripts/ui/home/HomeScreenController.ts`
- 首页壳层数据：
  - `assets/scripts/data/homeShell.ts`

还未创建：

- `assets/scenes/HomeScene.scene`

## 为什么这一步放在编辑器里做

当前项目还没有任何 `.scene` 文件。  
为了保持 Cocos 资源格式干净、兼容、标准，这一步不手写场景序列化文件，而是直接通过 Cocos Creator 编辑器创建。

## 接入步骤

### 1. 创建场景

在 Cocos Creator 中：

1. 在 `assets` 下新建文件夹 `scenes`
2. 在 `assets/scenes` 中新建场景：
   - 名称：`HomeScene`

## 2. 准备场景根结构

推荐保持最简结构：

- `HomeScene`
  - `Canvas`
  - `Main Camera`

如果 Cocos 已自动创建 `Canvas` 和 `Main Camera`，就直接复用，不要重复创建。

## 3. 挂载首页控制器

把脚本：

- `HomeScreenController`

挂到：

- `Canvas`

上。

## 4. 检查控制器参数

在 Inspector 中确认：

- `autoBuildRuntimeShell = true`

这一项保持开启。  
开启后，进入场景时会自动生成首页运行时壳层。

## 5. 运行后预期效果

进入场景后，应该看到：

1. 顶部资源栏
   - 金币
   - 点券
   - 章节
   - 战力

2. 主角主卡
   - 虾仁名字
   - 英雄口号
   - 主净化行动标题
   - 行动按钮

3. 当前章节主卡
   - 章节名
   - 危险等级
   - 场景说明
   - 战前建议

4. 2 x 2 快捷入口
   - 出战辅佐
   - 美女图鉴
   - 装备系统
   - 购买商城

5. 编组 / 图鉴摘要区

6. 底部导航条

## 6. 当前阶段允许的问题

这一版属于首页原型壳层，所以允许暂时存在：

- 仍然是程序化绘制块面，不是最终原画资源
- 点击入口只输出路由日志，不做正式跳转
- 还没有真正章节背景图和角色立绘

## 7. 当前阶段不应出现的问题

如果出现下面这些现象，说明要回头修：

- 场景报脚本挂载错误
- 首页结构没有生成
- 文本位置严重错乱
- 底部导航被顶出屏幕
- 手机上首屏内容超出可视区太多

## 8. 完成后建议动作

完成 `HomeScene` 接入后，下一步按这个顺序继续：

1. 用首页原型结构微调手机竖屏布局
2. 接入第一批真实原画资源
3. 再做 Formation / Codex / Shop 的首页跳转
