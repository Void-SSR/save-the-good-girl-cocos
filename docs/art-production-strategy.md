# 拯救美少女 Cocos 美术生产策略

## 目标

这份文档用于统一后续所有 Cocos 版美术资产的生产标准。目标不是“先把图做出来”，而是同时满足下面 4 个条件：

1. 角色与场景达到商用手游级别的原画观感
2. 角色魅力足够强，尤其是美女角色要有成熟、性感、曲线明确的视觉吸引力
3. 运行包体、首屏加载和内存占用必须可控，适合轻量化小游戏
4. 资源组织、导出和压缩流程必须标准化，能持续迭代，不靠临时手工凑

## 风格总则

### 角色风格

- 所有主要角色统一使用 `日系商业游戏原画` 方向
- 角色形体要明确区分：
  - 英雄：英俊、成熟、干练、轮廓利落
  - 美女：全部按成年角色设计，强调成熟魅力、身体曲线、服装层次和光影质感
  - Boss 黑化形态：在净化形态基础上做情绪和气场的极端化处理，强化压迫感
  - 净化形态：更明亮、更温润、更有角色好感度
- 禁止继续使用早期那种简单矢量图、扁平几何头像、低细节图标人物去承载主视觉

### 性感表达标准

- 方向是 `成熟、华丽、富有魅力`
- 重点通过这些要素去做：
  - 明确的头身比例
  - 清晰的腰臀胸部体块
  - 贴身或结构化服装剪裁
  - 腿部、肩颈、腰线的轮廓控制
  - 光影与材质表现带来的质感
- 不走低质擦边路线，不做露骨内容
- 所有美女统一保持“成人角色设定 + 商业原画审美”

### 场景风格

- 每个章节统一走 `恐怖、压抑、诡异但精致` 的场景原画方向
- 场景不是杂乱贴图堆砌，而是要有明确主题：
  - 霓虹废区
  - 寒晶回廊
  - 雷落穹塔
  - 樱雾神社
  - 蒸汽船坞
  - 砂海遗都
  - 月港歌剧院
  - 深海研究所
  - 星轨温室
  - 天穹圣城
- 战前预览图优先传达章节氛围，不强塞角色立绘

## 资源分层策略

后续资源统一分成 3 层，不把“原画母版”和“运行时资源”混在一起。

### L0：原画母版

用途：
- PSD / CLIP / Krita / 高分辨率 PNG 原始文件
- 用于二次裁切、改稿、宣传图、未来立绘升级

规则：
- 不直接作为游戏运行资源
- 不放进轻量化运行包
- 不建议长期堆在运行仓库里
- 统一放外部美术目录或专用归档位置

建议命名：
- `hero_main_v001.psd`
- `beauty_hiyori_purified_v001.psd`
- `beauty_hiyori_corrupted_v001.psd`
- `stage_neon_preview_v001.psd`

### L1：生产导出层

用途：
- 从原画母版导出的清洁中间产物
- 给程序、美术、UI 联调使用

规则：
- 已裁切、已定构图，但保留足够清晰度
- 可进入项目仓库，但数量要受控
- 作为运行时导出的上游来源

### L2：运行时资源层

用途：
- 真正进入 Cocos 包体和 Asset Bundle 的文件

规则：
- 一律按场景用途导出
- 一律按运行尺寸和内存预算压缩
- 不允许直接把大原画塞进运行包

## 角色资源规格

### 1. 首页 / 图鉴主卡角色图

用途：
- 首页主角卡
- 美女图鉴
- 招募结果页
- Boss 警报页

建议规格：
- 母版长边：`2800 ~ 3600 px`
- 运行时导出大图：`1024 x 1536` 左右
- 运行时导出中图：`768 x 1152`
- 缩略图：`320 x 480`

导出规则：
- 图鉴详情优先用中图
- 图鉴列表优先用缩略图
- Boss 警报优先用中图裁切版
- 首页只用主角单独裁切图，不加载完整角色库

### 2. Boss 黑化 / 净化双形态

每个主线美女至少要有：
- 净化立绘 1 张
- 黑化 Boss 立绘 1 张

允许：
- 共用结构草稿和姿态
- 但最终渲染必须拉开气质、颜色、表情、特效和服装细节差异

不允许：
- 直接简单滤镜换色当黑化版

### 3. 招募美女

要求：
- 每个角色轮廓明显不同
- 发型、服装、武器/配件、站姿都要区分
- 保证漂亮、成熟、有角色魅力

未解锁状态：
- 游戏内展示只用 `剪影版缩略图`
- 不泄露真实五官和服装细节

## 场景资源规格

### 战前场景预览

用途：
- 章节卡
- 战前预览
- 章节详情页

建议规格：
- 母版：`2560 x 1440` 或更高
- 运行时导出：`1280 x 720`
- 如需更轻量，再导出一个 `960 x 540` 备用版

### 战斗背景

建议不要把整张原画直接当一张超大图塞进战斗。

推荐结构：
- 背景远景层
- 中景建筑层
- 近景遮挡层
- 环境粒子层

优点：
- 更容易做轻量视差
- 单张图尺寸更容易控制
- 后续替换局部更方便

## UI 图标与小资源策略

### 图标

- 装备、护具、道具、技能、货币统一做成高质量小图标
- 图标基础画布建议：
  - `128 x 128`
  - 重要图标可保留 `256 x 256` 母版
- 运行时优先用 `128 x 128`

### 角色缩略图

- 图鉴列表缩略图建议：
  - `256 x 384`
  - 或 `320 x 480`
- 不要在列表里加载详情大图

## 文件格式策略

高兼容优先，不用不稳定的小众格式。

### 1. 大型不透明场景图

优先：
- `JPG`

建议：
- 质量区间 `82 ~ 88`
- 先看肉眼，再看体积

适用：
- 战前章节场景图
- 不需要透明通道的背景图

### 2. 角色立绘 / 需要透明通道的资源

优先：
- `PNG`

原因：
- 兼容最好
- Cocos / 小程序环境更稳

后续可选优化：
- 如果目标平台验证稳定，再评估批量转 `WebP`
- 但默认先以兼容为第一优先

### 3. 小图标

优先：
- `PNG`

### 4. 不建议当前阶段使用

- 未充分验证的平台特化压缩格式
- 需要额外运行时解码成本的复杂方案
- 为了省一点体积却显著增加兼容风险的格式

## 包体与加载预算

以下是当前项目建议目标，不是死规则，但后续资源制作要尽量围绕这个预算。

### 首屏首页资源预算

- 首页首屏总资源建议控制在：`5 MB` 以内
- 首页首次必须资源：
  - 主角图
  - 当前章节卡背景
  - 货币与核心 UI 图标
  - 极少量通用按钮图

不要首屏加载：
- 全部美女立绘
- 全部章节场景图
- 全部 Boss 大图
- 全部商城资源

### 单章节战前资源预算

- 当前章节预览图 + Boss 警报图：建议 `2 MB` 以内

### 单张角色运行时建议上限

- 图鉴详情图：尽量 `300 ~ 450 KB`
- 列表缩略图：尽量 `60 ~ 120 KB`
- Boss 警报裁切图：尽量 `180 ~ 320 KB`

## Cocos 资源组织建议

后续在 Cocos 中建议用 `Asset Bundle` 或等价分包思路管理资源：

- `common-ui`
  - 通用按钮、货币、基础图标
- `home`
  - 首页主角、首页章节卡、首页装饰
- `codex`
  - 图鉴缩略图、图鉴详情图
- `campaign`
  - 章节场景预览图
- `battle-common`
  - 通用怪物、英雄、通用特效
- `battle-stage-xx`
  - 某章节独有 Boss 图和场景层

原则：
- 首页只拉首页资源
- 图鉴进入后再加载图鉴资源
- 章节预览进入后再加载章节资源
- 战斗资源按章节拆，不全量预载

## 生产流程

后续每张资源建议按这个顺序生产：

1. 先定角色设定与姿态
2. 再出原画母版
3. 再导出 L1 清洁中间图
4. 再按用途裁出运行时大图 / 中图 / 缩略图
5. 再做压缩和体积检查
6. 最后接入 Cocos

禁止顺序：
- 先生成一张大图直接丢进项目再说
- 没有裁切规格就乱塞
- 运行图和原画母版混着用

## 质量验收标准

角色资源至少满足：
- 第一眼不再像占位图
- 五官、发型、服装、体块关系清楚
- 黑化和净化版一眼可区分
- 缩略图状态下也能认出角色轮廓

技术资源至少满足：
- 首页首屏不一次性加载全角色
- 图鉴列表不加载详情大图
- 场景预览不使用超大原图直塞
- 每张图都有用途，不留冗余废资源

## 当前执行结论

从这一刻开始：

- 主视觉角色全部改为原画路线
- 美女角色统一按成年、成熟、性感、商业原画标准设计
- 黑化与净化双形态必须独立完成
- 运行时资源必须经过裁切与压缩，不再直接使用简陋矢量图
- 后续无论是找资源还是生成资源，都必须先符合这份策略，再接入项目
