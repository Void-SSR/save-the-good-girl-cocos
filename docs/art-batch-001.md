# 首批原画资源清单

这份清单是 Cocos 版正式美术生产的第一批资源范围。  
目标不是一次把所有章节做完，而是先把首页、首章和首轮可视化验证需要的关键资源补齐。

## 批次目标

这一批只服务 3 个目标：

1. 让 `HomeScene` 从程序化块面进入“有正式游戏感”的阶段
2. 让第 1 章拥有可验证的角色和场景标准
3. 建立后续章节可复制的资源生产范式

## 本批必须资源

### 1. 主角首页立绘

- 资源 ID：`hero_xiaren_home_v001`
- 用途：
  - 首页主卡
  - 后续装备页主角立绘复用
- 风格要求：
  - 成熟、英俊、利落
  - 商业手游男主标准
  - 面部轮廓清晰，服装结构明确
- 原画母版建议：
  - `3000 x 4200`
- 运行时导出建议：
  - 大图：`768 x 1152 PNG`
  - 备选中图：`512 x 768 PNG`
- 包体要求：
  - 首页运行图尽量控制在 `250 ~ 380 KB`

### 2. 第 1 章净化美女立绘

- 资源 ID：`beauty_hiyori_purified_v001`
- 用途：
  - 图鉴
  - 编组
  - 招募展示
  - 结算净化展示
- 风格要求：
  - 成年、成熟、性感、有曲线感
  - 净化后要明亮、温暖、有舞台感
- 原画母版建议：
  - `3000 x 4200`
- 运行时导出建议：
  - 详情图：`768 x 1152 PNG`
  - 缩略图：`320 x 480 PNG`

### 3. 第 1 章黑化 Boss 立绘

- 资源 ID：`beauty_hiyori_corrupted_v001`
- 用途：
  - Boss 警报动画
  - 战前 Boss 预警页
  - 章节宣传图局部构图
- 风格要求：
  - 必须是独立黑化完成图
  - 不能只是净化版滤镜换色
  - 气质要强压迫感、危险感
- 原画母版建议：
  - `3000 x 4200`
- 运行时导出建议：
  - 警报裁切图：`768 x 1152 PNG`
  - 备选特写图：`640 x 960 PNG`

### 4. 第 1 章场景预览图

- 资源 ID：`stage_neon_preview_v001`
- 用途：
  - 首页当前章节卡
  - 章节列表
  - 战前预览
- 风格要求：
  - 恐怖、诡异、霓虹废区
  - 重点体现环境，不强塞角色脸
- 原画母版建议：
  - `2560 x 1440`
- 运行时导出建议：
  - 主图：`1280 x 720 JPG`
  - 轻量版：`960 x 540 JPG`
- 包体要求：
  - 主图尽量控制在 `220 ~ 360 KB`

### 5. 首页核心 UI 图标

必须补这几枚：

- `ui_icon_gold_v001`
- `ui_icon_ticket_v001`
- `ui_icon_formation_v001`
- `ui_icon_codex_v001`
- `ui_icon_equipment_v001`
- `ui_icon_shop_v001`

规格建议：

- 母版：`256 x 256`
- 运行时：`128 x 128 PNG`

要求：

- 风格统一
- 不能再使用早期临时几何块替代

## 运行时接入优先级

这批资源接入顺序建议固定为：

1. `hero_xiaren_home_v001`
2. `stage_neon_preview_v001`
3. `ui_icon_*`
4. `beauty_hiyori_purified_v001`
5. `beauty_hiyori_corrupted_v001`

原因：

- 首页最先需要主角和章节氛围
- 小程序首屏不能一次加载太多角色大图
- 第 1 章双形态美女可以在首页接入后再补

### 图鉴包

- 绯音净化立绘

### Boss 警报包

- 绯音黑化立绘

## 验收标准

本批完成后，应该达到：

1. 首页不再像程序占位图
2. 主角第一眼有正式男主感
3. 第 1 章场景图能单独传达章节氛围
4. 绯音净化 / 黑化形态一眼区分
5. 首屏加载仍然保持轻量，不因高质量图直接拖慢体验
